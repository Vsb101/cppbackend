Выполняем функции асинхронно — post
В прошлом уроке вы научились распределять работу между несколькими потоками и выполнять обработчики последовательно. Для этого вы использовали последовательный исполнитель strand. Он помогает защититься от состояния гонки без явных блокировок. Это позволяет потокам заниматься полезной работой вместо ожидания на мьютексе.
Кроме асинхронного ожидания таймеров, Boost.Asio позволяет асинхронно вызывать функции. Разберём на примере, что такое асинхронный вызов функции и чем он отличается от синхронного.
Допустим, ваш будний день начинается с завтрака, после которого вы моете посуду, вытираете стол и идёте на работу. Выполняя домашние дела одно за другим, вы можете опоздать на работу. Синхронно работающие программы устроены так же. 
При асинхронном подходе после завтрака вы заливаете тарелки водой, оставляете на холодильнике записку «Помыть посуду и прибраться на столе» и идёте на работу. Вечером вы видите записку и прибираетесь. Возможно, после вашего ухода записку обнаружит кто-то из домашних и приберётся на кухне за вас.
В этом примере работу выполняете вы и ваши близкие, а в программах — потоки. В синхронном примере один исполнитель последовательно выполняет задачи, а в асинхронном задачи помещаются в очередь, откуда их извлекает первый освободившийся исполнитель.
Для асинхронного выполнения функций в Boost.Asio есть функции post, defer и dispatch.
Функция post
Функция post (англ. отправить) помещает функцию, переданную ей в качестве аргумента, в очередь для асинхронного исполнения. Гарантируется, что переданная функция не будет выполнена на текущем потоке до возврата из post:

#include <boost/asio.hpp>
#include <iostream>
#include <thread>

namespace net = boost::asio;

int main() {
    net::io_context io;
    std::cout << "Eat. Thread id: " << std::this_thread::get_id() << std::endl;

    net::post(io, [] {
        std::cout << "Wash dishes. Thread id: " << std::this_thread::get_id() << std::endl;
    });

    net::post(io, [] {
        std::cout << "Cleanup table. Thread id: " << std::this_thread::get_id() << std::endl;
    });

    std::cout << "Work" << std::endl;
    io.run();
    std::cout << "Sleep" << std::endl;
} 

В этой программе лямбда-функция, переданная в post, будет помещена в очередь внутри контекста io и выполнится в момент вызова io.run(). Так как в программе только один поток, посуда будет помыта, а стол прибран в основном потоке. Но уже после возвращения с работы. Асинхронные операции выполнятся в том же порядке, в каком они попали в очередь.

Eat. Thread id: 13420
Work
Wash dishes. Thread id: 13420
Cleanup table. Thread id: 13420
Sleep 

Функция-обработчик отработает в потоке, вызвавшем метод io.run(). Это может быть поток, отличный от того, который поместил функцию в очередь:

int main() {
    net::io_context io;
    std::cout << "Eat. Thread id: " << std::this_thread::get_id() << std::endl;

    net::post(io, [] {
        std::cout << "Wash dishes. Thread id: " << std::this_thread::get_id() << std::endl;
    });

    net::post(io, [] {
        std::cout << "Cleanup table. Thread id: " << std::this_thread::get_id() << std::endl;
    });

    std::cout << "Work" << std::endl;
    std::jthread{[&io] {
        io.run();
    }}.join();
    std::cout << "Sleep" << std::endl;
} 

Так как io.run() вызван в другом потоке, мытьё посуды и приборка на столе будут выполнены в нём, как если бы на кухне прибрался кто-то из домашних:

Eat. Thread id: 284
Work
Wash dishes. Thread id: 2516
Cleanup table. Thread id: 2516
Sleep 

Если потоков, которые вызывают io.run(), несколько, работа распределится между ними. Например, кроме мытья посуды и стола можно разделить и другие домашние дела:

#include <syncstream>

template <typename Fn>
void RunWorkers(unsigned n, const Fn& fn) { /* Функция из предыдущего урока */}

int main() {
    using osync = std::osyncstream;

    net::io_context io;
    std::cout << "Eat. Thread id: " << std::this_thread::get_id() << std::endl;

    net::post(io, [] {  // Моем посуду
        osync(std::cout) << "Wash dishes. Thread id: " << std::this_thread::get_id() << std::endl;
    });

    net::post(io, [] {  // Прибираемся на столе
        osync(std::cout) << "Cleanup table. Thread id: " << std::this_thread::get_id() << std::endl;
    });

    net::post(io, [&io] {  // Пылесосим комнату
        osync(std::cout) << "Vacuum-clean room. Thread id: " << std::this_thread::get_id()
                         << std::endl;

        net::post(io, [] {  // После того, как пропылесосили, асинхронно моем пол
            osync(std::cout) << "Wash floor. Thread id: " << std::this_thread::get_id()
                             << std::endl;
        });

        net::post(io, [] {  // Асинхронно опустошаем пылесборник пылесоса
            osync(std::cout) << "Empty vacuum cleaner. Thread id: " << std::this_thread::get_id()
                             << std::endl;
        });
    });

    std::cout << "Work. Thread id: " << std::this_thread::get_id() << std::endl;

    RunWorkers(2, [&io] {  // Асинхронные операции выполняются двумя потоками
        io.run();
    });
    std::cout << "Sleep" << std::endl;
} 

Мытьё пола и очистка пылесборника запускаются в обработчике, который пылесосит комнату. Поэтому пол будет помыт, а пылесборник очищен после уборки пылесосом:

Eat. Thread id: 5716
Work. Thread id: 5716
Wash dishes. Thread id: 24008
Cleanup table. Thread id: 5716
Vacuum-clean room. Thread id: 24008
Empty vacuum cleaner. Thread id: 24008
Wash floor. Thread id: 5716
Sleep 

Задачи распределяются между двумя потоками случайно. Это может меняться от запуска к запуску, зависеть от ОС, активности других программ и от конфигурации компьютера. Чтобы задачи не выполнялись слишком быстро и не обрабатывались одним потоком, можно добавить паузы внутрь лямбда-функций. Для этого используйте std::this_thread::sleep_for.
В качестве исполнителя можно указать не только io_context, но и strand. Функции внутри одного strand выполнятся последовательно в том порядке, в котором они были отправлены на выполнение функцией post. Это позволяет избежать состояния гонки без явного использования примитивов синхронизации. 
В примере ниже последовательность вызова функций будет фиксированной, даже если сами функции станут выполняться в разных потоках:

int main() {
    net::io_context io{2};
    auto strand = net::make_strand(io);
    int value = 10;

    net::post(strand, [&value, &strand] {  // Эта функция будет первой внутри strand
        value += 20;                       // (1)

        net::post(strand, [&value] {  // Этот post поставит функцию третьей по счёту
            value += 10;                   // (3)
        });
    });

    net::post(strand, [&value, &strand] {  // Эта функция будет второй внутри strand
        value *= 40;                       // (2)

        net::post(strand, [&value] {  // Этот post поставит функцию четвёртой по счёту
            value *= 20;                   // (4)
        });
    });

    RunWorkers(2, [&io] {
        io.run();
    });

    assert(value == ((10 + 20) * 40 + 10) * 20);
} 

Укажите порядок, в котором будут выведены надписи в этой программе.

int main() {
    net::io_context io;

    net::post(io, [&io] {
        std::cout << "Read book" << std::endl;

        net::post(io, [] { std::cout << "Write essay" << std::endl; });
    });

    net::post(io, [] { std::cout << "Play football" << std::endl; });

    std::cout << "Watch movie" << std::endl;
    io.run();
    std::cout << "Study English" << std::endl;
} 

Как вам задача?
Функции, которые переданы в post, будут поставлены в очередь внутри io_context, а не выполнены немедленно. Поэтому сначала будет выведена надпись “Watch movie”. 
Метод io.run() будет извлекать функции из очереди и выполнять их в порядке FIFO. От англ. first in, first out — «первым пришёл, первым ушёл». Принцип, при котором данные обрабатываются в порядке их поступления на обработку, как очередь в супермаркете, т.е.:

    Вызывается функция, которая выводит “Read book”. Перед выходом она добавляет в очередь функцию, которая выводит “Write essay”.
    Вызывается функция, которая выводит “Play football”.
    Вызывается функция, которая выводит “Write essay”.

После этого io.run() завершает работу, так как других функций в очереди не осталось. Последней выведется надпись “Study English”.
Укажите все возможные варианты выводимых строк в этой программе.

int main() {
    using osync = std::osyncstream;

    net::io_context io{2};
    auto strand = net::make_strand(io);

    net::post(strand, [] { osync(std::cout) << "Cat, "; });

    net::post(strand, [] { osync(std::cout) << "Fish, "; });

    net::post(io, [] { osync(std::cout) << "Dog, "; });

    RunWorkers(2, [&io] {
        io.run();
    });

    std::cout << std::endl;
} 

Тоже правильный ответ
Cat, Fish, Dog,
Правильный ответ
Cat, Dog, Fish,
Fish, Cat, Dog,
Fish, Dog, Cat,
Правильный ответ
Dog, Cat, Fish,
Dog, Fish, Cat,
Как вам задача?
В этой программе вывод строки “Cat” идёт всегда перед выводом “Fish”, так как располагается раньше внутри strand. Вывод строки “Dog” добавлен в io_context напрямую.

Функции вывода кота и рыбы привязаны к strand, а функция вывода собаки — нет
Если бы io.run() вызывался в единственном потоке, вывод был бы "Cat, Fish, Dog, ". Но так как потока два, одновременно за вывод могут начать бороться строки “Cat“ и “Dog“. Если победит “Dog“, вывод будет “Dog, Cat, Fish, ”. Если победит “Cat”, то за второе место будут бороться “Fish” и “Dog”, что даст ещё два возможных исхода: “Cat, Fish, Dog, ” и “Cat, Dog, Fish, ”.

Выполняем функции асинхронно — defer
В прошлом уроке мы рассмотрели функцию post для асинхронного исполнения функций.
Функция boost::asio::defer, как и post, помещает функцию в очередь для асинхронного выполнения. post информирует другие потоки о появлении функции в очереди, чтобы первый освободившийся поток начал её выполнять. defer этого не делает, если вызывающий код уже выполняется контекстом — например, вызван из метода io_context::run.
Так выше вероятность, что асинхронная функция выполнится в потоке, вызвавшем defer. Если асинхронная функция продолжает текущую, например, работает с теми же данными, отложенный вызов может сделать использование кэша эффективнее. Если асинхронная функция не продолжает вызывающую, использование defer вместо post может привести к тому, что функция будет вызвана позже, чем могла бы.
Это можно сравнить с опытом человека, который собирается купить себе кухонную мебель. Он приходит в магазин мебели, где один из менеджеров делает ему предварительный расчёт стоимости. Потом через какое-то время человек решает, что хочет заказать кухню именно в этой фирме. Будет эффективнее отправить клиента к тому же менеджеру, который делал изначальный расчет и помнит подробности заказа.
То, как в действительности будет распределяться работа между потоками, зависит от контекста, которому адресован вызов defer. Например, для io_context нет разницы между defer и post. В Boost.Asio есть также пул потоков, который распределяет задачи между заданным множеством потоков — thread_pool. Он не будет выполнять задачи, поставленные в очередь функцией defer, до завершения текущей.

int main() {
    using osync = std::osyncstream;
    using namespace std::chrono;

    // Создаём пул, содержащий два потока
    net::thread_pool tp{2};
    const auto start = steady_clock::now();

    auto print = [start](char ch) {
        const auto t = duration_cast<duration<double>>(steady_clock::now() - start).count();
        osync(std::cout) << std::fixed << std::setprecision(6) 
            << t << "> " << ch << ':' << std::this_thread::get_id() << std::endl;
    };

    net::post(tp, [&tp, print] {
        print('A');

        net::defer(tp, [print] { print('B'); });
        net::defer(tp, [print] { print('C'); });
        net::defer(tp, [print] { print('D'); });

        // Засыпаем, чтобы дать шанс другим потокам сделать свою работу
        std::this_thread::sleep_for(std::chrono::seconds(1));
    });

    // Дожидаемся окончания работы потоков
    tp.wait();
} 

В этом примере функции символы B, C и D будут выведены спустя одну секунду после вывода символа A, только после завершения лямбда-функции.

0.000057> A:18964
1.011520> C:14348
1.011774> D:14348
1.011510> B:18964 

❗ В одном потоке с символом A выведен лишь символ B. Символы С и D выведены во втором потоке.
Теперь заменим defer на post:

int main() {
    using osync = std::osyncstream;
    using namespace std::chrono;

    // Создаём пул, содержащий два потока
    net::thread_pool tp{2};
    const auto start = steady_clock::now();

    auto print = [start](char ch) {
        const auto t = duration_cast<duration<double>>(steady_clock::now() - start).count();
        osync(std::cout) << std::fixed << std::setprecision(6) 
            << t << "> " << ch << ':' << std::this_thread::get_id() << std::endl;
    };

    net::post(tp, [&tp, print] {
        print('A');

        // Теперь вместо defer используется post
        net::post(tp, [print] { print('B'); });
        net::post(tp, [print] { print('C'); });
        net::post(tp, [print] { print('D'); });

        // Засыпаем, чтобы дать шанс другим потокам сделать свою работу
        std::this_thread::sleep_for(std::chrono::seconds(1));
    });

    // Дожидаемся окончания работы потоков
    tp.wait();
} 

Поведение программы изменилось. Символы B, C и D будут выведены до окончания выхода из лямбда-функции. Пока один из потоков спал, второй вывел эти символы.

0.000013> A:18992
0.000230> B:12552
0.000296> C:12552
0.000360> D:12552 

Поведение defer зависит от контекста, который выполняет асинхронные функции. Заменим thread_pool на io_context:

int main() {
    using osync = std::osyncstream;
    using namespace std::chrono;

    net::io_context io{2};
    const auto start = steady_clock::now();

    auto print = [start](char ch) {
        const auto t = duration_cast<duration<double>>(steady_clock::now() - start).count();
        osync(std::cout) << std::fixed << std::setprecision(6) 
            << t << "> " << ch << ':' << std::this_thread::get_id() << std::endl;
    };

    net::post(io, [&io, print] {
        print('A');

        net::defer(io, [print] { print('B'); });
        net::defer(io, [print] { print('C'); });
        net::defer(io, [print] { print('D'); });

        // Засыпаем, чтобы дать шанс другим потокам сделать свою работу
        std::this_thread::sleep_for(std::chrono::seconds(1));
    });

    RunWorkers(2, [&io]{ io.run(); });
} 

Запустим программу и увидим, что в этом случае разницы между post и defer нет: символы B, C и D выведены вторым потоком, пока первый спал.

0.000459> A:10304
0.000598> B:20676
0.000708> C:20676
0.000800> D:20676 

Итак, вызов defer говорит исполнителю: «Эту функцию надо выполнить асинхронно в указанном контексте. Желательно — в этом же потоке, так как она продолжает работу текущей асинхронной функции». Прислушаться к требованиям или нет — решает исполнитель.
В аналогии с мебельной фирмой клиент мог обратиться в компанию, в которой при повторном обращении клиента ему предоставят первого свободного менеджера, а не того, с которым клиент взаимодействовал в первый раз. В разных компаниях могут быть разные бизнес-процессы.

Выполняем функции асинхронно — dispatch
Третий способ выполнить функцию асинхронно — функция boost::asio::dispatch. post и defer всегда вызывают функцию асинхронно. dispatch работает иначе: он выполняет функцию синхронно, если вы вызываете dispatch внутри этого же исполнителя.

int main() {
    net::io_context io;

    // dispatch поместит вызов лямбда-функции в очередь, так как dispatch вызван вне io.run()
    net::dispatch(io, [&io] {
        std::cout << "Work hard" << std::endl;

        // post всегда выполняет функцию асинхронно
        net::post(io, [] {
            std::cout << "Have dinner" << std::endl;
        });

        // Этот dispatch вызовет лямбда функцию синхронно, так как dispatch вызван внутри io.run
        net::dispatch(io, [] {
            std::cout << "Eat an apple" << std::endl;
        });

        std::cout << "Go home" << std::endl;
    });

    std::cout << "Go to work" << std::endl;
    io.run();
    std::cout << "Sleep" << std::endl;
} 

Здесь первый вызов dispatch расположен вне io.run(), поэтому функция, выводящая “Work hard”, будет поставлена в очередь, и не вызовется до вызова io.run(). 
Функция вывода “Eat an apple” выполняется внутри io.run(), поэтому будет вызвана немедленно. Этим поведение отличается от функции, вызванной через post, Напомним, что post всегда ставит функцию в очередь. Программа выведет:

Go to work
Work hard
Eat an apple
Go home
Have dinner
Sleep 

dispatch вызовет функцию синхронно, только если переданный в dispatch исполнитель связан с текущим контекстом исполнения.
Посмотрите на пример ниже. В функции, которая исполняется в контексте io1 , дважды вызывается dispatch:

    с контекстом io2,
    со strand, созданным в контексте io1:

int main() {
    net::io_context io1;
    net::io_context io2;
    auto strand = net::make_strand(io1);

    net::dispatch(io1, [&io2, &strand] {
        // Эта функция будет вызвана в контексте io1
        std::cout << "Enter io1 lambda" << std::endl;
        net::dispatch(io2, [] {  // Эта функция будет вызвана асинхронно в контексте io2
            std::cout << "Execute in io2" << std::endl;
        });
        net::dispatch(strand,
                      [] {  // Эта функция будет вызвана синхронно внутри strand в контексте io1
                          std::cout << "Execute in strand" << std::endl;
                      });
        std::cout << "Leave io1 lambda" << std::endl;
    });

    std::cout << "io1.run()" << std::endl;
    io1.run();
    std::cout << "io2.run()" << std::endl;
    io2.run();
} 

Надпись “Execute in io2” выведется только после входа в io2.run(). Дело в том, что функция dispatch(io2, ...), вызванная в контексте io1, поставит вызов в очередь внутри контекста io2. А вот dispatch(strand, ...) вызовет функцию синхронно, так как последовательный исполнитель strand связан с контекстом io1,  внутри которого мы находимся. Программа выведет:

io1.run()
Enter io1 lambda
Execute in strand
Leave io1 lambda
io2.run()
Execute in io2 

int main() {
    net::io_context io;

    net::post(io, [&io] {  // (1)
        std::cout << 'A';
        net::post(io, [] {  // (2)
            std::cout << 'B';
        });
