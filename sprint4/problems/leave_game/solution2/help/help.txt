Добавляем БД в игру
Разберёмся, как работать с базой данных в многопоточном приложении.
Чтобы обмениваться информацией с сервером базы данных, библиотека libpqxx предоставляет класс pqxx::connection. Этот класс берёт на себя сетевое взаимодействие с сервером. Класс pqxx::transaction позволяет атомарно выполнять последовательность запросов к серверу с помощью идиомы RAII. В своей работе он использует pqxx::connection.
И pqxx::transaction, и pqxx::connection — обычные классы, одновременная работа с которыми из разных потоков может приводить ошибкам, зависаниям или краху приложения. Документация рекомендует рассматривать каждый экземпляр pqxx::connection вместе со всеми связанными с ним объектами в виде некого «‎мира». Если какой-либо из потоков программы выполняет над «‎миром» модифицирующие операции, из других потоков к этому «‎миру» обращаться нельзя.
Это задание — заключительное для вашего проекта. После его выполнения вам надо будет сдать программу на ревью.

Как вам задача?
Ваш игровой сервер использует несколько потоков. Безопасно использовать в нём одно соединение с БД возможно, только если транзакции будут выполняться строго последовательно. Если все обращения к API сервера, а также шаги игровых часов всегда выполняются внутри одного strand, одного соединения может быть достаточно для безопасной работы. Но если в будущем вы захотите обрабатывать события, относящиеся к разным картам, в отдельных потоках, единственное соединение с БД может ограничить производительность вашего сервера.
Установление нового соединения с базой данных всякий раз, когда потребуется выполнить запрос, может негативно сказаться на производительности. Соединение с сервером может занимать сотни миллисекунд, тогда как выполнение транзакции — считанные миллисекунды.

int main() {
    using namespace std::chrono;
    try {
        const char* db_url = std::getenv("DB_URL");
        if (!db_url) {
            throw std::runtime_error("DB URL is not specified");
        }
        const auto start_time = steady_clock::now();
        steady_clock::time_point conn_time;
        steady_clock::time_point tx_construction_time;
        steady_clock::time_point query_end_time;
        steady_clock::time_point tx_end_time;
        steady_clock::time_point conn_end_time;
        {
            pqxx::connection conn{db_url};
            conn_time = steady_clock::now();
            {
                pqxx::read_transaction tx{conn};
                tx_construction_time = steady_clock::now();
                std::ignore = tx.query1<int>("SELECT 1;");
                query_end_time = steady_clock::now();

            tx_end_time = steady_clock::now();
        }
        conn_end_time = steady_clock::now();

        std::cout << "Connection time: "
                  << duration_cast<duration<double>>(conn_time - start_time).count() << std::endl;
        std::cout << "Create transaction time: "
                  << duration_cast<duration<double>>(tx_construction_time - conn_time).count()
                  << std::endl;
        std::cout << "Query time: "
                  << duration_cast<duration<double>>(query_end_time - tx_construction_time).count()
                  << std::endl;
        std::cout << "Destroy transaction time: "
                  << duration_cast<duration<double>>(tx_end_time - query_end_time).count()
                  << std::endl;
        std::cout << "Close connection time: "

Выходом в этой ситуации будет пул соединений — заранее подготовленный набор соединений, из которого программа их получает. Пример такого пула соединений приведён ниже.

class ConnectionPool {
    using PoolType = ConnectionPool;
    using ConnectionPtr = std::shared_ptr<pqxx::connection>;

public:
    class ConnectionWrapper {
    public:
        ConnectionWrapper(std::shared_ptr<pqxx::connection>&& conn, PoolType& pool) noexcept
            : conn_{std::move(conn)}
            , pool_{&pool} {
        }

        ConnectionWrapper(const ConnectionWrapper&) = delete;
        ConnectionWrapper& operator=(const ConnectionWrapper&) = delete;

        ConnectionWrapper(ConnectionWrapper&&) = default;
        ConnectionWrapper& operator=(ConnectionWrapper&&) = default;

        pqxx::connection& operator*() const& noexcept {
            return *conn_;
        }
        pqxx::connection& operator*() const&& = delete;

        pqxx::connection* operator->() const& noexcept {
            return conn_.get();
        }

        ~ConnectionWrapper() {
            if (conn_) {
                pool_->ReturnConnection(std::move(conn_));
            }
        }

    private:
        std::shared_ptr<pqxx::connection> conn_;
        PoolType* pool_;
    };

    // ConnectionFactory is a functional object returning std::shared_ptr<pqxx::connection>
    template <typename ConnectionFactory>
    ConnectionPool(size_t capacity, ConnectionFactory&& connection_factory) {
        pool_.reserve(capacity);
        for (size_t i = 0; i < capacity; ++i) {
            pool_.emplace_back(connection_factory());
        }
    }

    ConnectionWrapper GetConnection() {
        std::unique_lock lock{mutex_};
        // Блокируем текущий поток и ждём, пока cond_var_ не получит уведомление и не освободится
        // хотя бы одно соединение
        cond_var_.wait(lock, [this] {
            return used_connections_ < pool_.size();
        });
        // После выхода из цикла ожидания мьютекс остаётся захваченным

        return {std::move(pool_[used_connections_++]), *this};
    }

private:
    void ReturnConnection(ConnectionPtr&& conn) {
        // Возвращаем соединение обратно в пул
        {
            std::lock_guard lock{mutex_};
            assert(used_connections_ != 0);
            pool_[--used_connections_] = std::move(conn);
        }
        // Уведомляем один из ожидающих потоков об изменении состояния пула
        cond_var_.notify_one();
    }

    std::mutex mutex_;
    std::condition_variable cond_var_;
    std::vector<ConnectionPtr> pool_;
    size_t used_connections_ = 0;
}; 

Перед использованием этого класса надо создать пул соединений нужного размера. В конструктор ConnectionPool передаётся функция, которую пул вызовет для создания каждого из запрошенных соединений. Эта функция может подготовить соединение с сервером — например, создать подготовленные запросы. Размер пула  можно выбрать равным количеству потоков компьютера.
После конструирования пула из него можно получать свободные соединения методом GetConnection. Если соединений нет, метод заблокирует текущий поток, пока не появятся свободные соединения. Метод GetConnection возвращает прокси-объект ConnectionPool::ConnectionWrapper, который в своём деструкторе вернёт соединение обратно в пул.
Класс ConnectionPool — потокобезопасный. С ним можно безопасно работать в нескольких потоках одновременно. Для синхронизации доступа к своим данным класс использует классы [std::mutex] и [std::condition_variable].
Полученный экземпляр ConnectionWrapper можно безопасно применить в одном из потоков, чтобы использовать хранящийся в нём объект pqxx::connection. Все объекты, которые используют полученное соединение, должны быть разрушены до разрушения ConnectionWrapper, так как после этого соединение вернётся в пул и станет доступным другим потокам.
Заменим предыдущий пример на использование пула потоков:

int main() {
    using namespace std::chrono;
    try {
        const char* db_url = std::getenv("DB_URL");
        if (!db_url) {
            throw std::runtime_error("DB URL is not specified");
        }
        const auto start_time = steady_clock::now();
        ConnectionPool conn_pool{10, [db_url] {
                                     auto conn = std::make_shared<pqxx::connection>(db_url);
                                     conn->prepare("select_one", "SELECT 1;");
                                     return conn;
                                 }};
        auto pool_creation_time = steady_clock::now();
        steady_clock::time_point conn_time;
        steady_clock::time_point tx_construction_time;
        steady_clock::time_point query_end_time;
        steady_clock::time_point tx_end_time;
        steady_clock::time_point conn_end_time;
        {
            auto conn = conn_pool.GetConnection();
            conn_time = steady_clock::now();
            {
                pqxx::read_transaction tx{*conn};
                tx_construction_time = steady_clock::now();
                std::ignore = tx.exec_prepared1("select_one").as<int>();
                query_end_time = steady_clock::now();
            }
            tx_end_time = steady_clock::now();
        }
        conn_end_time = steady_clock::now();

        std::cout << "Pool creation time: "
                  << duration_cast<duration<double> >(pool_creation_time - start_time).count()
                  << std::endl
                  << "-------------" << std::endl;
        std::cout << "Getting connection time: "
                  << duration_cast<duration<double> >(conn_time - pool_creation_time).count()
                  << std::endl;
        std::cout << "Create transaction time: "
                  << duration_cast<duration<double> >(tx_construction_time - conn_time).count()
                  << std::endl;
        std::cout << "Query time: "
                  << duration_cast<duration<double> >(query_end_time - tx_construction_time).count()
                  << std::endl;
        std::cout << "Destroy transaction time: "
                  << duration_cast<duration<double> >(tx_end_time - query_end_time).count()
                  << std::endl;
        std::cout << "Close connection time: "
                  << duration_cast<duration<double> >(conn_end_time - tx_end_time).count()
                  << std::endl;
        std::cout << "Total time: "
                  << duration_cast<duration<double> >(conn_end_time - pool_creation_time).count()
                  << std::endl;
    } catch (const std::exception& e) {
        std::cerr << e.what() << std::endl;
    }
}

Как вам задача?
Эта программа создаёт пул потоков, содержащий одно соединение. Сперва программа получит единственное соединение и сохранит его в переменную conn1. При попытке получить второе соединение, программа будет ждать, пока в пуле не появится свободное соединение. Но этого не произойдёт, так как переменная conn1 не разрушится до выхода из функции main. А ход выполнения не вернётся в функцию main, пока в пуле не появится свободное соединение. Такая ситуация называется взаимной блокировкой или дедлоком.
Возможные способы защититься от такой ситуации:

    Получить соединение лишь на короткое время и в один момент времени использовать только одно соединение в потоке.
    Доработать класс ConnectionPool, чтобы он выбрасывал исключение, если поток, который получил соединение, пытается получить новое, не вернув старое.
    Ограничить время ожидания соединения некоторым разумным значением. Если за отведённое время получить соединение из пула не удалось, выбросить исключение.
    Получать соединение из пула асинхронно.

Чтобы не переусложнять вашу программу, рекомендуем получать единственное соединение лишь на ограниченный промежуток времени.
